## pac CLI on-premises support — consolidated note

**pac CLI does not support on-premises Dynamics 365 CE.** This is confirmed explicitly in two Microsoft sources:

- A Microsoft maintainer on the build-tools GitHub discussions (Dec 2024): "PAC cli is not supported for connecting to on premises deployments. It is coded to support only online environments." (https://github.com/microsoft/powerplatform-build-tools/discussions/1097)
- The `pac modelbuilder` docs state: "The Power Platform CLI isn't available for Dynamics 365 Customer Engagement on-premises." (https://learn.microsoft.com/en-us/power-apps/developer/data-platform/org-service/generate-early-bound-classes)

**Exceptions — commands that work offline / without a live auth profile:**

- `pac solution pack` and `pac solution unpack` operate on the local filesystem only — no auth needed, work for on-prem pipelines.
- `pac solution version` (local XML/YAML edits only, no auth needed).
- `pac solution create-settings` (reads a local ZIP, no auth needed).
- `pac tool prt/cmt/pd/maker/admin` — launches GUI tools; `pac tool` itself is Windows + .NET Full Framework only.

All other command groups (`pac auth`, `pac admin`, `pac solution export/import/publish/sync/check`, `pac plugin push`, `pac pcf push`, `pac data`, `pac env`, `pac modelbuilder`, `pac pcf`, `pac pipeline`, `pac canvas`, `pac connector`, `pac copilot`, `pac pages`, `pac managed-identity`) require a live connection to Dataverse online (Entra ID / OAuth) and will error against an on-prem NTLM or IFD target.
