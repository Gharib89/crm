# Harvest 03 — XrmToolBox catalogue

**Family:** Most-installed XrmToolBox plugins — each popular plugin exists because a recurring task is painful in the native UI; rows derive the underlying task. XrmToolBox connects via the Dataverse SDK and works on-prem and online unless noted.
**Collected:** 2026-06-10 by research subagent (read-only web research).

| Scenario | Persona | Frequency-signal | Pain-signal | Tooling-today | Source URL |
|---|---|---|---|---|---|
| Write and register a plugin step against an entity event (both) | pro-code | **high** — server-side business logic is the primary code artifact on every D365 project | **high** — compile → register → test loop has no hot-reload; wrong stage/image name causes silent failures; sandbox restrictions differ between on-prem (full-trust) and online | Plugin Registration Tool (PRT) or `pac plugin push`; XrmToolBox Plugin Registration (637k DLs) | https://www.solvingdynamics365.com/guides/plug-in-registration-tool-deep-dive |
