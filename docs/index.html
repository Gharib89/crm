§03Recipes from the field

tried on MOCE 9.1.44.15

ABuild a custom table, end to end

# 1. Create the table, primary attr, and auto-publish.
crm --json metadata create-entity \
  --schema-name new_Project \
  --display "Project" \
  --display-collection "Projects" \
  --primary-attr new_ProjectName \
  --primary-label "Project Name" \
  --ownership UserOwned \
  --solution DevSandbox

# 2. Confirm it landed.
crm metadata entity new_project

# 3. Insert a row.
crm --json entity create new_projects \
  --data-json '{"new_name":"Apollo"}'

BMass-extract a view to CSV

crm data export contacts \
  --output contacts.csv \
  --select fullname,emailaddress1,telephone1 \
  --filter "statecode eq 0" \
  --page-size 500 \
  --max-records 10000 \
  --format csv

CLookups & associations

crm entity set-lookup \
  contacts <contactid> parentcustomerid_account \
  accounts <accountid>

DFire an on-demand workflow

# 1. Find the workflow id.
crm --json workflow list \
  --entity contact --on-demand --activated

# 2. Run it against a contact.
crm --json workflow run <workflowid> \
  --target <contactid>

# Or via the generic bound-action escape hatch:
crm action invoke ExecuteWorkflow \
  --bind-set workflows --bind-id <workflowid> \
  --body '{"EntityId":"<contactid>"}'

§04Still only on the wire

honest gaps · raw OData required

Anything below is reachable today by hand-rolling against the Dynamics Web API, but has no first-class verb yet. The CLI's action invoke and action function escape hatches cover most actions; metadata mutations need the underlying POST/PATCH/DELETE on EntityDefinitions(…).

AreaWhat the API can do todayHow to reach it now
P0Delete / update entity Drop a custom table, rename labels, toggle HasActivities/HasNotes after creation. Raw DELETE /EntityDefinitions(LogicalName='x') or PATCH. CLI verb missing — cleanup of test tables on MOCE today required manual auth.
P0Attribute CRUD Add new columns of any type — string, picklist, lookup, money, datetime — and edit or drop them. POST /EntityDefinitions(LogicalName='x')/Attributes with the appropriate @odata.type cast.
P1Relationships Create 1:N, N:1, and N:N relationships incl. cascade rules and intersect entities. POST /RelationshipDefinitions with the OneToMany or ManyToMany cast.
P1Global option sets Create / update / delete GlobalOptionSetDefinitions and tie picklists to them. POST /GlobalOptionSetDefinitions; CLI today only reads via metadata picklist.
P1Batch operations Multi-statement transactions via $batch with changesets — atomic multi-row inserts. Raw HTTP POST /$batch with multipart/mixed. No verb yet.
DONEWorkflows · v0.2 List, activate, deactivate, and run on-demand workflows. Shipped as workflow list / activate / deactivate / run. BPF (category=4) and Modern Flow (category=5) records are visible via --category.
P2Security model Roles, teams, business units, privileges, principal access (RetrievePrincipalAccess). CRUD over roles, teams, businessunits via entity works; principal-access function via action function.
P2Files & images Upload / download File and Image column payloads, chunked uploads. Raw PATCH/GET on {set}({id})/{column}/$value with x-ms-file-name header. No verb yet.
P3Bulk operations BulkDelete, BulkDetectDuplicates, ImportSolutionAsync, AsyncOperation polling. action invoke for the request; status polling must be hand-rolled.
P3Plugins & SDK steps Register assemblies, plugin types, steps, images; manage service endpoints. CRUD over pluginassemblies, plugintypes, sdkmessageprocessingsteps via entity; the upload itself needs custom encoding.
P3Audit & telemetry Read audit records, RetrieveRecordChangeHistory, system jobs. query odata on audits; history via action function RetrieveRecordChangeHistory.
DONEBound actions · v0.2 Call actions bound to a specific record, e.g. workflows(id)/ExecuteWorkflow. Shipped: action invoke <Name> --bind-set workflows --bind-id <guid> (--cast overrides namespace).
P3Raw HTTP override Issue PUT / DELETE calls against arbitrary OData paths (e.g. metadata cleanup). No --method switch on action invoke yet; the bound/unbound action path only POSTs.

The shortest path to closing every gap above: ship metadata delete-entity, metadata create-attribute, and a --method escape hatch on action invoke. Three commits, ninety percent of the wishlist. // Roadmap · v0.2

§05Design notes

why it bites this way

01JSON envelope, every time

With --json every command emits {ok, data, meta, error}. Agents and humans parse the same payload. meta.count, meta.session, meta.dry_run come for free.

02Backslashes never sourced

set -a; . ./.env mangled moce\crmadmin into mocecrmadmin. The CLI's own load_dotenv reads the file directly so the domain backslash survives.

03$top client-side

v9.1 on-prem rejects $top against EntityDefinitions with an explicit 400. metadata entities --top N slices the response after retrieval.

04Auto-publish after metadata writes

metadata create-entity --publish chains PublishAllXml so a new table is immediately visible to UI consumers. Opt out with --no-publish when bundling several changes.

05Dry-run for everything

The global --dry-run flag short-circuits every write — POST, PATCH, DELETE, action dispatch — and returns the would-be payload. Use it as a credential-free unit test.

06Tests are not optional

47/47 green: 39 unit + 8 live E2E against MOCE 9.1.44.15. CLI_ANYTHING_FORCE_INSTALLED=1 requires the binary on PATH so CI never falls back to python -m.
